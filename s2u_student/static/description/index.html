Teacher Student Content

share your videos on a secure way with your students or people you want. Provide it as a e-learning page or as a 1 on 1 lesson (private lesson).

You need Vimeo!

To work with this module you need a pro Vimeo account (or higher)

Register App in Vimeo

In Vimeo you need to register your app.

Scopes you must enable: Public, Private, Create, Edit, Delete, Interact and Upload. The scope "Upload" you need to request by Vimeo, it takes some days before this scope is reviewed by Vimeo and released ...

Go to Vimeo developer to register your app.

You need the following keys in your Odoo System parameters:

  • s2u_student.vimeo.client_token
  • s2u_student.vimeo.client_folder (optional)
  • s2u_student.vimeo.whitelist_domain (optional)

The Token you get from the app you created in Vimeo. The Folder is optional, when set all videos you add in Odoo are uploaded in this folder. This way you can organize your content when you are sharing your Vimeo account over multiple Odoo instances.

For using a Folder, create a folder first in vimeo and open it. In the url you see something like this: https://vimeo.com/manage/folders/1495089

You need to use the id on the end (1495089) as value for s2u_student.vimeo.client_folder

Important: use domain for whitelist

Set the key "whitelist_domain" with the domain your are hosting your Odoo environment. This way videos can only be played from this domain. If your are not setting this key,'localhost' is used by default.

In the Student module you can add new content. Give it a title, description and define which groups/users (portal) have access to this content. When target is "e-Learning" then the content is accesable over the url "/student/elearning". When target is "Personal", you need to select a single student and a predefined Project. For this video a new task is created and is accesable in the portal account of the Student. The teacher and student can use the Odoo chatter for communicating about this task.

When all data is entered correctly and you selected your video to share, save and hit the button "Publish content". Odoo will now push the content to vimeo. Our module is using the "pull" method from Vimeo, meaning Vimeo will request the video over a specific url our module is providing to Vimeo. Therefore it is important your are running this module on a system accessable over the internet.

As soon Vimeo has pulled the video, the state changes from "Synchronizing" into "Published". Also a cover image is provided automatically when you did not provide one in the first place.

Visit your content over the url "/student/elearning" to see if everything is working correctly.
